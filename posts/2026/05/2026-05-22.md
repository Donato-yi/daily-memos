# Daily Memo - 2026-05-22

## TL;DR
- Agent-first development is becoming a platform concern, not just an IDE feature.
- Runtime maintenance releases still deserve careful rollout: Node LTS carried crypto/certificate updates, while Python 3.14.5 reverted its incremental GC after memory pressure reports.
- The practical engineering theme today is controlled change: compare drift, preserve rollback paths, and move security checks closer to where code is written.

## Tech News & Notes
- Google's I/O 2026 developer keynote centered on agent orchestration: Gemini 3.5, Antigravity 2.0, an Antigravity CLI, and specialized subagents for complex developer workflows.
- Node.js 22.22.3 LTS shipped with operationally relevant maintenance, including a crypto null-pointer fix, updated root certificates, timezone data, and dependency updates.
- Python 3.14.5 shipped about 154 fixes and reverted the 3.14 incremental garbage collector back to the generational GC from 3.13 after production memory-pressure reports.
- The Rust project closed its 2025H2 project-goals period, reporting on 41 goals and carrying selected flagship work forward into the 2026 period.
- GitHub is rolling out a new format for newly minted GitHub App installation tokens. Apps that validate token shape with brittle regexes need to loosen those checks before the rollout reaches them.

## Concerns / Risks
- Agent platforms need explicit trust boundaries: sandboxing, credential masking, audit logs, repo policy enforcement, and controlled delegation.
- LTS runtime updates can still affect production behavior through crypto, certificate, timezone, and dependency changes. They should move through the same smoke tests as larger upgrades.
- Token format changes are easy to underestimate. Anything that assumes fixed token prefixes or lengths can fail even when authentication itself is working correctly.

## Memorials / Significant Events
- Google I/O 2026 is the significant developer event this week. The headline is not just better models; it is the shift toward agent orchestration as a standard developer platform primitive.

## What I Learned Today
- Baseline comparison is more useful than raw violation counts for architecture fitness checks. A team needs to know what was introduced, fixed, and left unchanged to make the next review decision.
- Security and architecture work both benefit from direction-of-travel reporting. A small regression is easier to act on when the tool can say precisely what changed since the last known state.
