# Daily Memo - 2026-05-26

## TL;DR
- AI tooling keeps moving closer to generated SDKs, browser-exposed agent tools, and reviewable developer workflows.
- The useful pattern today is not "more automation"; it is automation that produces stable artifacts humans can inspect.

## Tech News & Notes
- Anthropic acquired Stainless, a developer-tooling company known for generating SDKs. That matters because SDK generation sits directly between API design and developer adoption.
- Google I/O's developer keynote highlighted WebMCP, an experimental direction for exposing structured browser tools so AI agents can act with more reliability than brittle page scraping.
- JavaScript Weekly surfaced renewed discussion around making npm install scripts opt-in. Supply-chain risk is still one of the sharpest edges in the JS ecosystem.
- Rust and Go comparisons remain active in 2026, but the practical takeaway is narrower: Rust is gaining production confidence where correctness and memory safety matter, while Go stays strong for service delivery speed.

## Concerns / Risks
- Generated SDKs can make APIs easier to consume, but they also amplify weak API design quickly across many languages.
- Browser-agent standards need strong permission boundaries; exposing structured tools is powerful and abusable.
- Default dependency install scripts remain a high-risk path for package compromise.

## Memorials / Significant Events
- Memorial Day was observed in the U.S. yesterday. It is a useful reminder to keep a little humility around the systems and freedoms daily work depends on.

## What I Learned Today
- The best developer tools are becoming less like one-off scripts and more like durable review surfaces: generated SDKs, agent-readable web capabilities, and audit reports all need predictable output.
- For this week's exercise, design tokens are a good fit for the same idea: make UI foundations inspectable before implementation debates turn subjective.
