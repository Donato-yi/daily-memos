# Daily Memo — 2026-05-13

## TL;DR
- GitHub is pushing security checks earlier into AI-assisted coding via MCP secret and dependency scanning.
- GitHub also says agentic development load forced a reliability plan shift from 10x to 30x scale.
- Google Cloud is packaging agent building, governance, and deployment into a broader Gemini Enterprise platform.
- ChatGPT’s May updates focused on better memory sourcing, a faster default model, and richer answers for free users.

## Tech News & Notes
- GitHub made secret scanning in the GitHub MCP Server generally available for repos with Secret Protection enabled. The notable bit is policy continuity: push protection customizations now carry through the MCP flow, which makes AI-assisted checks feel more like a first-class part of the secure SDLC instead of a sidecar tool.
- GitHub also put dependency scanning for the MCP Server into public preview. That means an MCP-compatible coding agent can flag vulnerable packages before commit time, with affected packages, severity, and recommended fixed versions coming back as structured results.
- GitHub’s availability update is the bigger platform story. They said October 2025 planning assumed a 10x capacity increase, but by February 2026 they realized they needed to design for 30x current scale because agentic workflows are accelerating repo creation, pull requests, APIs, and automation load together.
- At Google Cloud Next, Google introduced Gemini Enterprise and the Gemini Enterprise Agent Platform as an evolution of Vertex AI. The pitch is a single place to build, govern, optimize, and operate fleets of agents, with ADK, MCP support, connectors, and stronger IT oversight built in.
- OpenAI’s ChatGPT release notes for May 2026 highlight memory sources for more transparent personalization, GPT-5.5 Instant becoming the default model, trusted contact rollout for supported personal accounts, and more inline web images for free users.

## Concerns / Risks
- Moving security checks earlier into coding agents is great, but noisy or slow pre-commit scans can get bypassed if they interrupt developer flow too often.
- GitHub’s 30x scaling comment is a reminder that agentic development is not just a model problem. It is a systems, queueing, cache, and blast-radius problem across the entire platform.
- Enterprise agent platforms are converging on governance-heavy narratives, which is probably necessary, but teams still need simpler local-first patterns so experimentation does not get trapped behind platform complexity.

## Memorials / Significant Events
- May 2026 feels like a real shift from “AI can assist coding” to “AI-driven workflows are changing how source control, security review, and platform reliability are designed.”

## What I Learned Today
- The most interesting GitHub announcements were not flashy model features. They were operational: pre-commit security controls for agents and a candid description of how agentic usage is reshaping infra planning.
- The agent-platform race is increasingly about orchestration plus governance, not just raw model quality.
- Transparency features like memory sources matter because personalization gets much more useful once users can inspect where it came from.
