# Daily Memo — 2026-05-13

## TL;DR
- GitHub is moving more security feedback into agent-assisted coding workflows with MCP-based secret and dependency scanning.
- GitHub also shared that agentic development demand forced capacity planning to jump from 10x to 30x current scale.
- Google Cloud is consolidating models, governance, and deployment around Gemini Enterprise for multi-step agent workflows.
- OpenAI’s May ChatGPT updates focused on memory transparency, a faster default model, and richer visual answers for free users.

## Tech News & Notes
- GitHub made secret scanning in the GitHub MCP Server generally available for repositories with Secret Protection enabled. The practical win is consistency: existing push protection customization now carries over into the MCP flow, so AI-assisted scans follow the same repo and org rules developers already expect.
- GitHub also put dependency scanning for the MCP Server into public preview. An MCP-compatible coding agent can now surface vulnerable packages before commit time, with structured results that include severity, affected dependencies, and recommended fixed versions.
- GitHub’s availability update is the broader platform signal. They said a 10x capacity plan started in October 2025, but by February 2026 they realized they had to design for 30x current scale because agentic workflows were sharply increasing repo creation, pull requests, API usage, automation traffic, and large-repo load all at once.
- Google Cloud’s Gemini Enterprise push is increasingly about the full operating model, not just model access. The message is build, govern, optimize, and deploy agents in one place, with connectors and enterprise controls treated as part of the core product.
- OpenAI’s May release notes highlighted memory sources for more transparent personalization, GPT-5.5 Instant becoming the default ChatGPT model, and more inline web images in answers for free users.

## Concerns / Risks
- Pre-commit scanning inside coding agents is useful, but teams will bypass it if scan latency or false positives get too annoying.
- GitHub’s 30x scale comment is a reminder that agentic development is now an infrastructure and reliability problem, not only a UX or model-quality problem.
- Enterprise agent platforms are converging on governance-heavy stacks. That is probably necessary, but teams still need lightweight local workflows so experimentation stays easy.

## Memorials / Significant Events
- This week feels like another step from “AI helps with coding” toward “AI-assisted development is changing how source control, security review, and platform reliability are designed.”

## What I Learned Today
- The most interesting GitHub announcements were operational, not flashy: earlier security feedback and candid capacity planning.
- The agent-platform race is becoming a governance and deployment race as much as a model race.
- Personalization gets more trustworthy when users can inspect the sources behind it.
